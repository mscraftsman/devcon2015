Media Queries

They do not magically make your website work on various devices!


You have to apply your CSS accordingly...

Viewport

The Viewport meta tag was first introduced by Apple, the adopted and developed further by others


When visiting a website on a mobile browser, it will assume that you're viewing a big desktop experience and that you want to have it all displayed as it is on the desktop, it therefore automatically set the viewport width at 980px (taking iOS safari as en example), trying to fit everything on its small screen.

Viewport width should be set to width of device!

Using media queries 

					
		body{ background: #fff; color: #000;}
		
		@media screen and (max-width: 768px){
			body{ background: #ff0000; }
		}
		
		@media screen and (max-width: 320px){
			body{ background: #0000ff; }
		}

For screens 768px - 320px, the background colour will be red. It overides the first fule background: #fff


Similarly, for screens 0-320px, the background colour will be blue, overidding the two previous rules.

Awesomeness

There's more to responsive web design than just some media queries

Fluid Grid Layout

Regardless of what devices or screen, components in fluid designs are going to flow and adapt to the environment.

Fluid grids flow naturally, limited adjustments will be needed for various screen sizes.

We cannot plan for only Small devices in responsive design. So with fluid grid layout we can play with the max-width and it will still work on larger screens.
